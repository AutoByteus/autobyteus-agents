# Character Registry

## Registry Rule

- Keep one row per canonical character id.
- Update this file when a character is introduced, renamed, redesigned, retired, killed, or brought back.
- Use this as the fast lookup layer for long-running series work. Detailed psychology and design notes still belong in `character-bible.md`.
- This file is the series-level authority for mutable character status fields such as debut chapter, latest appearance, roster status, design-lock version, current voice mapping, and reusable asset refs.

## Status Guidance

- `planned`: exists as a deliberate future character, not fully production-ready yet
- `active`: currently available for normal story use
- `inactive`: exists but is offstage for now
- `hidden`: identity exists but should not be broadly surfaced in spoiler-free contexts
- `retired`: no longer active in the story but still part of continuity
- `dead`: canonically dead unless later canon changes that
