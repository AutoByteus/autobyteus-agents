# Resume Style Briefs

Use these pure-text briefs as starting points for dynamic app generation. They describe design intent and constraints; they are not fixed code templates, fixed DOM structures, or mandatory component layouts. Adapt each brief to the user's actual content, target role, and hiring context before writing frontend code.

Default restraint rule: even expressive briefs should still look like resumes, not landing pages, dashboards, portfolios, or marketing one-pagers. Use typography, spacing, and reading order as the main design tools. Use color and structure sparingly. Prefer one quiet separator system; never place a dark header rule directly above a light section rule.

## Executive Classic

- Best for: senior leadership, operations, finance, consulting, legal, enterprise sales
- Layout grammar: usually one-column, strong name block, compact section dividers, restrained whitespace
- Typography: serious serif or neutral sans pairing
- Color: black, charcoal, one conservative accent such as navy or deep green
- Avoid: playful icons, bright color, dense sidebars
- PDF risk: low
- ATS risk: low

## Technical Minimal

- Best for: software engineering, data, security, infrastructure, technical leadership
- Layout grammar: one-column or light two-column header, dense experience bullets, clear skills taxonomy
- Typography: neutral sans with optional monospace accents for technologies
- Color: black, slate, one muted accent such as blue or teal
- Separators: subtle gray section rules are usually enough; avoid heavy header rules unless there is no adjacent section divider
- Avoid: skill bars, unnecessary charts, overloaded icon rows, duplicate horizontal rules
- PDF risk: low
- ATS risk: low

## Product Builder

- Best for: product management, founder/operator, growth, design-adjacent technical roles
- Layout grammar: strong but resume-like headline, selected impact highlights, project or product section with scannable outcomes
- Typography: modern sans with confident hierarchy
- Color: dark neutral plus one muted accent used sparingly
- Separators: keep boundaries quiet and singular; the founder/product highlight should feel like a resume section, not a product card
- Avoid: marketing-landing-page composition, oversized hero sections, dashboard panels, boxed signal grids, duplicate separators, vague claims
- PDF risk: medium if spacing gets too expressive
- ATS risk: low to medium

## Design Portfolio Resume

- Best for: UX, UI, brand, creative direction, frontend/design hybrid roles
- Layout grammar: visually distinctive but printable, optional project cards, clear portfolio link emphasis
- Typography: refined sans or editorial pairing
- Color: controlled palette with strong contrast
- Avoid: image-only layouts, decorative elements that crowd the content, tiny type
- PDF risk: medium
- ATS risk: medium unless semantic text remains simple

## Academic Research

- Best for: academic, research science, labs, fellowships, publications-heavy candidates
- Layout grammar: CV-like sections, publications and education prominent, conservative spacing
- Typography: serif or academic neutral sans
- Color: black and grayscale with minimal accent
- Avoid: corporate sales summary language, excessive compression of publications
- PDF risk: low
- ATS risk: low

## Early Career Clean

- Best for: students, interns, recent graduates, career switchers
- Layout grammar: education and projects can lead; skills grouped clearly; one-page bias
- Typography: clean sans with generous but controlled spacing
- Color: neutral plus one accessible accent
- Avoid: pretending sparse experience is senior, large empty visual blocks
- PDF risk: low
- ATS risk: low

## Creative Editorial

- Best for: writing, content, media, arts, selective creative roles
- Layout grammar: editorial hierarchy, strong summary, selected work section
- Typography: editorial serif/sans pairing
- Color: restrained palette; accent can be more expressive than corporate briefs
- Avoid: overly decorative print artifacts, low contrast, portfolio visuals replacing resume text
- PDF risk: medium
- ATS risk: medium

## Conservative ATS

- Best for: online applications, recruiter databases, regulated industries, unknown audience
- Layout grammar: single column, conventional section names, dense but readable bullets
- Typography: common sans or serif, no unusual symbols
- Color: black and grayscale
- Avoid: sidebars, icons, columns for core facts, nonstandard headings
- PDF risk: very low
- ATS risk: very low
